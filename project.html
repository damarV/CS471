Team 01 - User Services and Systems Rubicon
  • Microservices
    • Auth & Federation
      • Cognito User Pools for signup/signin, MFA, passwordless
      • Cognito Identity Pools for IAM-backed identities for apps
      • API Gateway + Lambda Authorizer for JWT/token enforcement
      • Amazon Verified Permissions for fine-grained auth (Cedar)
      • Route 53 + ACM + CloudFront for secure auth domains
    • User Profiles & Storage
      • DynamoDB (or Aurora Serverless v2) for profile records
      • S3 for avatars, attachments; S3 Object Lambda for transforms
      • API Gateway + Lambda for profile CRUD APIs
      • DynamoDB Streams + Lambda for reactive updates
    • Roles, Orgs & Permissions
      • Cognito Groups / Custom Claims for student/faculty/admin
      • Verified Permissions for ABAC/RBAC policy decisions
      • EventBridge for role/permission change events
    • Sessions & Device Trust
      • Cognito for token lifetimes, refresh, revocation
      • CloudFront + ALB (if needed) for session-aware edges
      • DynamoDB for device registry & session blacklist
      • Lambda@Edge (optional) for header/cookie gates
    • Notifications & Preferences
      • SNS for fan-out events to services
      • SES / Pinpoint for email & SMS; preference center
      • DynamoDB for user comms preferences
      • EventBridge for “UserCreated”, “RoleChanged”, etc.
    • Directory/SSO Integration (Campus)
      • Cognito Federation for SAML/OIDC to campus IdP
      • Lambda + Step Functions for nightly roster sync jobs
      • SQS DLQ for resilient sync pipeline
    • Admin Console & Self-Service
      • CloudFront + S3 for static admin SPA hosting
      • API Gateway + Lambda for admin APIs (suspend, reset MFA)
      • Verified Permissions for admin UI policy guardrails
